## Architecture Refactor Decision (2026-04-10 afternoon)

### Root Cause of Pipeline Fragility
- One massive SKILL.md (~1000 lines) for the watch-listing agent
- LLM doing ALL work: JSON writes, math, formatting, substitutions, file ops
- By Phase B the context is so full the agent stalls before writing anything
- draft_save.py partial fix: prevents corrupt JSON, atomic writes
- But Phase B still times out loading reference files before writing first platform

### Approved Architecture (to build and test incrementally)
Split the monolith into: tight LLM micro-skills + Python for everything deterministic

**Python tools (no LLM needed):**
- `draft_save.py` ← already built (deep merge, atomic write, validates JSON)
- `run_pricing.py` ← all platform price math from inputs
- `run_phase_b.py` ← assembles full listing .md from canonical + templates + substitutions
- `run_grailzee_gate.py` ← web fetch + comparison (maybe)
- `generate_listing_pdf.py` ← already built

**LLM micro-skills (synthesis/judgment only):**
- `step0-watchtrack/SKILL.md` ← browser nav + data extraction (50 lines max)
- `step1-photos/SKILL.md` ← visual assessment + condition notes (50 lines max)
- `step3a-canonical/SKILL.md` ← write canonical (reads voice-tone.md, one para + one sentence)

**Orchestrator:**
- `pipeline.py` ← reads draft, routes to right tool or skill, no LLM logic

**Build order:** run_phase_b.py → run_pricing.py → micro-skills → pipeline.py

### New Canonical Description Format
- One descriptive paragraph (horological significance + what makes this reference worth owning)
- One sentence only for condition (not a full paragraph)
- Voice-tone.md read fresh at start of Phase A — separate file, mnemo caches it
- Tier rules updated to match: all tiers = 1 paragraph + 1 sentence

### voice-tone.md
- Built at: `watch-listing-workspace/skills/watch-listing/references/voice-tone.md`
- Contains: Ogilvy standard, Voss standard, knowledge vs confidence tier, hard rules, tier rules, platform notes, canonical format
- Agent reads this fresh at start of Phase A — always at top of context, cached by mnemo

### draft_save.py
- Built at: `watch-listing-workspace/tools/draft_save.py`
- Usage: `python3 draft_save.py <folder_path> '<json_patch>'`
- Deep merges patch into existing draft, validates, writes atomically (.tmp then rename)
- Backs up corrupt .json to .corrupt.bak before overwriting
- Agent must NEVER write _draft.json directly — always call this tool
