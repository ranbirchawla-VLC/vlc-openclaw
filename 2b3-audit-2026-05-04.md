## 11. Gaps

Sourced from `agent_api_integration_pattern.md` and `AGENT_ARCHITECTURE.md`. No fixes proposed.

- **`tools.allow` missing GTD tools.** Six registered tools (`capture_gtd`, `query_tasks`, `query_ideas`, `query_parking_lot`, `review_gtd`, `delegation`) are not in the agent entry `tools.allow`. Source: `agent_api_integration_pattern.md` §"Register in openclaw.json" — `tools.allow` must list `<tool_a>, <tool_b>, message`; `AGENT_ARCHITECTURE.md` §3 (plugin registration) — tool not in allow is not surfaced to LLM. Confirmed by live session tool list.

- **No `startActiveSpan` wrapping in `index.js`.** `execute()` calls `spawn()` without a tracer span; no `TRACEPARENT` injected via `SPAWN_ENV`. Source: `agent_api_integration_pattern.md` §"index.js: plugin entry with OTEL wrapping" — "Every plugin uses this shape. `startActiveSpan` wrapping is mandatory." `AGENT_ARCHITECTURE.md` locked pattern 2 — "missing parent context attachment is a build-breaking gap."

- **No `SPAWN_ENV` constant with `OTEL_SERVICE_NAME`.** `index.js` passes `env: { ...process.env }` directly instead of a `SPAWN_ENV` module constant. Source: `agent_api_integration_pattern.md` §"index.js" — `SPAWN_ENV = { ...process.env, OTEL_SERVICE_NAME: PLUGIN_TRACER }`.

- **No `@opentelemetry/api` dependency in `package.json`.** The pattern requires `"dependencies": { "@opentelemetry/api": "^1.9.0" }` for `trace`, `context`, `propagation` imports. `gtd-tools/package.json` has no `dependencies` key. Source: `agent_api_integration_pattern.md` §"package.json".

- **AGENTS.md does not match the AGENT_ARCHITECTURE skeleton.** Missing: Hard Rules block, correct Tools Available list by registered name, prohibition on exec/read/write/edit/browser, On Every Startup or PREFLIGHT block. Present content (pipeline.md reference, exec/read/write listed as tools) contradicts locked patterns 3 and 4. Source: `AGENT_ARCHITECTURE.md` §"AGENTS.md skeleton".

- **SKILL.md not at workspace root.** Gateway injects `<workspace>/SKILL.md`; no such file exists. The file at `skills/gtd/SKILL.md` is not loaded. Agent has no capability instructions in production. Source: `AGENT_ARCHITECTURE.md` §"Directory layout" — `~/.openclaw/workspace/skills/<agent>/SKILL.md`.

- **Mislocated SKILL.md references non-existent v1 tools.** Content refers to `gtd_normalize`, `gtd_write`, `gtd_review`, `gtd_validate`, and LLM skill files (`llm_ambiguous_classifier.md` etc.) that do not exist in the current plugin or on disk. Even if relocated to workspace root, it would provide incorrect capability instructions. Source: `agent_api_integration_pattern.md` §"Scope" — production SKILL.md must match the current registered tool surface.

- **`auth-profiles.json` missing from `agentDir`.** Source: `AGENT_ARCHITECTURE.md` §"Setup checklist" — "Create `auth-profiles.json`: `echo '{"version":1,"profiles":{}}' > ...`".
